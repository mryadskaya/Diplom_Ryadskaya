Дипломная работа "РАЗРАБОТКА ПРОГРАММНОГО МОДУЛЯ ДЛЯ ОБМЕНА ДАННЫМИ МЕЖДУ ИНФОРМАЦИОННОЙ СИСТЕМЫ 1С:ПРЕДПРИЯТИЕ И ЭЛЕКТРОННОЙ ИНФОРМАЦИОННО-ОБРАЗОВАТЕЛЬНОЙ СРЕДЫ УНИВЕРСИТЕТА НА БАЗЕ ФГАОУ ВО"СЕВЕРО - КАВКАЗСКИЙ ФЕДЕРАЛЬНЫЙ УНИВЕРСИТЕТ, Г. СТАВРОПОЛЬ""
1. Описание проекта
Программный модуль предназначен для обеспечения кроссплатформенного обмена данными между информационными системами «1С:Предприятие» и уже существующей Интегрированной автоматизированной системой управления (ИАСУ) в условиях перехода на операционную систему Alt Linux.

Модуль реализован на встроенном языке «1С:Предприятие», работает внутри кластера 1С и полностью заменяет устаревшую COM-компоненту. Он обеспечивает:

автоматический сбор изменённых приказов из плана обмена «ОбъектМетаданных»;

формирование SQL-запросов для вызова хранимых функций ИАСУ;

выполнение запросов через внешний источник данных (подключение к готовой ИАСУ);

поддержку Windows (через COM) и Linux (через прямой SQL через внешний источник данных);

создание соответствий между элементами справочников 1С и идентификаторами ИАСУ.

Модуль не имеет графического интерфейса, работает в фоновом режиме и не требует изменений в существующей ИАСУ.

3. Функциональность
3.1 Сбор изменённых приказов из плана обмена
Модуль выгружает изменения из плана обмена «ОбъектМетаданных». При проведении приказа ссылка на объект метаданных автоматически регистрируется в плане обмена. Функция СписокИзмененныхОбъектов выполняет запрос к таблице изменений и возвращает массив ссылок приказов, подлежащих выгрузке в ИАСУ.

3.2 Получение движений приказа
Для каждого приказа выполняется сложный запрос, который собирает движения приказа из различных таблиц («ОбщиеДанные», «СписокПриказов», «ПереходыСостоянийФизическихЛиц» и др.). Результат возвращается в виде таблицы значений, содержащей все необходимые поля для вызова хранимых процедур ИАСУ.

3.3 Формирование SQL-запроса
Для каждой строки движений формируется текст SQL-запроса. С помощью процедуры ПодставитьПараметрЗапросаSQL в запрос подставляются параметры с приведением к нужным типам (дата, строка, число, булево значение, NULL).

3.4 Запись движений в ИАСУ (кроссплатформенная логика)
В зависимости от операционной системы сервера 1С используется разный механизм:

Для Windows:

Создаётся COM-объект ADODB.Connection.

Создаётся COM-объект ADODB.Command, ему передаётся текст запроса.

Команда выполняется, результат преобразуется в таблицу значений.

Для Linux:

Текст запроса анализируется, определяется имя хранимой процедуры.

Параметры извлекаются и преобразуются в массив.

Выполняется прямая SQL-команда через внешний источник данных (подключение к готовой ИАСУ).

Особый случай для UNI_Kartapromat_Update:
Данная процедура имеет отличную от других структуру запроса (параметр передаётся как JSON-подобная строка). Для неё применяется особый алгоритм парсинга через поиск подстроки '(.

3.6 Контроль корректности записи
После выполнения INSERT-операции модуль выполняет SELECT-запрос к СУБД ИАСУ через внешний источник данных для подтверждения успешной записи. Для каждой процедуры предусмотрена своя функция-запрос:

Запрос_UNI_prikaz

Запрос_UNI_contingent_Insert

Запрос_UNI_MoveCont_Insert

Запрос_UNI_people_Insert

Запрос_UNI_sprgu_Insert / Запрос_UNI_sprgu_Select

Запрос_UNI_sprpgr_Insert_Select

3.7 Преобразование типов параметров
Функция ПреобразоватьСтрокиВНужныеТипы преобразует строковые значения параметров из текста SQL-запроса в корректные типы данных (числа, даты, булевы значения, строки, NULL) в зависимости от ожидаемой сигнатуры хранимой процедуры.

3.8 Создание соответствий между справочниками
После успешной записи движения модуль создаёт или обновляет запись в регистре сведений «Соответствие справочников информационных систем». Функция СоздатьСоответствиеЭлементовСправочников связывает идентификатор элемента в ИАСУ (kod_card, userID и др.) с внутренней ссылкой 1С. Это обеспечивает возможность последующего обновления данных без дублирования.

3.9 Логирование ошибок
При возникновении ошибок (недоступность COM, ошибка подключения к БД, синтаксическая ошибка SQL) информация об ошибке регистрируется через ВспомогательныеФункцииСервер.ЗарегистрироватьОшибкуВыполнения. При работе на Linux транзакции откатываются, если они активны.

5.2 Использование существующей ИАСУ для записи движений
При вызове хранимых процедур модуль обращается к таблицам и функциям существующей ИАСУ через настроенный внешний источник данных. Никакие объекты внутри ИАСУ не создаются и не модифицируются — выполняются только вызовы уже существующих хранимых функций (UNI_prikaz_Insert, UNI_contingent_Insert, UNI_people_Insert и др.).

Таким образом, модуль является только клиентом для готовой ИАСУ и не требует внесения изменений в её структуру.

6. Особенности реализации для Linux
COM-технология не используется. Вместо неё применяется подключение к существующей ИАСУ через внешний источник данных.

Разбор текста запроса. Имя хранимой процедуры и параметры извлекаются из строки SQL.

Особая логика для UNI_Kartapromat_Update. Поиск параметра по подстроке '(.

Управление транзакциями. При ошибке выполняется откат активной транзакции.

7. Безопасность
Аутентификация при подключении к ИАСУ осуществляется через параметры строки подключения (UID, PWD).

Параметры SQL-запроса подставляются с экранированием через процедуру ПодставитьПараметрЗапросаSQL.

Для проверки существования связи используется предварительный SELECT-запрос.

Логирование ошибок не содержит чувствительных данных (пароли не фиксируются).

8. Ограничения и особенности
Для работы на Linux требуется настроенный внешний источник данных (ODBC + FreeTDS), подключённый к существующей ИАСУ.

Для работы на Windows требуется доступ к COM-объектам ADODB.

ИАСУ не создаётся и не изменяется — модуль только подключается к уже готовой системе.

Хранимые функции ИАСУ не изменяются.

Все изменения в конфигурации 1С минимальны и ограничиваются настройкой внешнего источника данных и заменой COM-вызовов.

Модуль не имеет графического интерфейса и работает в фоновом режиме.

10. Краткий итог
Разработанный программный модуль полностью заменяет COM-компоненту и обеспечивает корректный обмен данными между системами 1С и уже существующей ИАСУ при переходе СКФУ на операционную систему Alt Linux. Модуль работает внутри кластера 1С, автоматически собирает изменённые приказы из плана обмена, формирует SQL-запросы и выполняет их через внешний источник данных, подключённый к готовой ИАСУ. Решение не требует изменения существующих хранимых функций ИАСУ, поддерживает работу как на Windows, так и на Linux, и готово к промышленной эксплуатации.
